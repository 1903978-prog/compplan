# Setup — compplan

Full-stack TypeScript app: Express + React/Vite + Drizzle ORM on PostgreSQL (Neon in production), with Anthropic Claude for AI generation. Deployed on Render as service `compplan`.

Modules: HR planning, pricing analysis, AI-powered proposal generation (slides + PPTX decks), hiring pipeline, Harvest invoice sync.

### Required
| Variable | Purpose | Used in | Fallback |
|---|---|---|---|
| `DATABASE_URL` | PostgreSQL connection string | `server/db.ts:11`, `drizzle.config.ts:12` | **None — throws if missing** |

### Optional (feature-gated)
| Variable | Purpose | Used in | If missing |
|---|---|---|---|
| `ANTHROPIC_API_KEY` | Claude API for proposal analysis, slide briefs, deck generation | `server/proposalAI.ts:150`, `server/proposalBriefs.ts:495/632/798`, `server/routes.ts:560/640/689` | AI endpoints return mock data with a console warning |

| `HARVEST_TOKEN` | Bearer token for Harvest invoicing API | `server/routes.ts:1244` | Harvest sync disabled |
| `HARVEST_ACCOUNT_ID` | Harvest `Harvest-Account-Id` header | `server/routes.ts:1245` | Harvest sync disabled |
| `PORT` | HTTP listen port | `server/index.ts:60` | `5000` (Render overrides dynamically) |
| `NODE_ENV` | `development` enables Vite HMR; `production` serves static bundle | `server/index.ts:53`, `script/build.ts:57` | Production behavior |

## 4. Authentication

Simple cookie-session auth — no JWT, Passport, or OAuth.

- Cookie: `compplan_session` = `authenticated`
- Flags: `httpOnly`, `secure` in prod, `sameSite=lax`, `maxAge=30 days`
- Endpoints (`server/auth.ts`):
  - `POST /api/auth/login` — body `{password}`, compared against `APP_PASSWORD`
  - `GET /api/auth/check` — returns `{authenticated}`
  - `POST /api/auth/logout` — clears cookie
- `requireAuth` middleware protects all data routes
- **If `APP_PASSWORD` is unset, everything is open** — intentional for local dev

API cost kill-switch (`/api/api-pause`): un-pausing accepts hardcoded `"1"`, `API_UNPAUSE_PASSWORD`, or `APP_PASSWORD`.

## 5. Database

- **Engine:** PostgreSQL (Neon in production)
- **ORM:** Drizzle (`drizzle-orm` + `drizzle-kit`)
- **Schema:** `shared/schema.ts` (~650 lines, ~21 tables)
- **Migrations:** `./migrations/` (generated by `npm run db:push`)
- **Seed:** `server/seed.ts` runs automatically on startup (`server/index.ts:38`). Uses `CREATE TABLE IF NOT EXISTS`; inserts default role grid (11 roles), 6 test employees, 7 tests, and template proposals from `seedProposals.ts`.

### Tables by module

**HR / People:** `employees`, `role_grid`, `salary_history`, `days_off_entries`, `employee_tasks`, `performance_issues`, `time_tracking_topics`, `time_tracking_entries`

**Pricing & Proposals:** `pricing_settings`, `pricing_cases`, `pricing_proposals`, `proposals` (includes `slide_selection`, `slide_briefs`, `preview_html`), `proposal_templates`, `slide_methodology_configs`, `deck_template_configs`, `project_type_slide_defaults`

**Hiring:** `hiring_candidates`

**Finance / Invoicing:** `invoice_snapshots`, `invoice_changes`

**Admin / Ops:** `app_settings`, `users` (legacy, unused), `api_usage_log` (Claude cost tracking)

## 6. AI Features

All Anthropic, model `claude-sonnet-4-20250514`:

| Endpoint | File | What it does |
|---|---|---|
| `POST /api/proposals/:id/analyze` | `server/proposalAI.ts` | Turn company + transcript into 3 proposal options, team, scope, deliverables |
| `POST /api/proposals/:id/generate-briefs` | `server/proposalBriefs.ts` | Per-slide content briefs using methodology configs |
| `POST /api/proposals/:id/slides/:slide_id/generate` | `server/proposalBriefs.ts` | Generate content for one specific slide |
| `POST /api/proposals/:id/generate-quick` | `server/routes.ts:952` | Analyze + brief in one call |
| `POST /api/proposals/:id/generate-deck` | `server/routes.ts:1027` | Build PPTX from templates |

- Token/cost logging: `logApiUsage()` at `server/routes.ts:33-47` writes to `api_usage_log`.
- Pricing constants hardcoded at `server/routes.ts:31` ($3/M input, $15/M output).
- System prompts: mostly hardcoded in TS files. Deck generation uses a DB-stored prompt in `deck_template_configs.system_prompt`, editable from the admin UI.

## 7. Build / Run / Deploy

### Scripts (`package.json`)
| Script | Command | Use |
|---|---|---|
| `npm run dev` | `NODE_ENV=development tsx server/index.ts` | Local dev with Vite HMR |
| `npm run build` | `tsx script/build.ts` | Vite → `dist/public/`, esbuild → `dist/index.cjs` |
| `npm start` | `NODE_ENV=production node dist/index.cjs` | Production |
| `npm run check` | `tsc` | Type-check only |
| `npm run db:push` | `drizzle-kit push` | Apply schema to DB |

### Runtime
- Node 20.x
- Listens on `0.0.0.0:${PORT||5000}`

### Render
- No Dockerfile — Render uses the Node buildpack
- **Build command:** `npm install && npm run build`
- **Start command:** `npm start`
- Render injects `PORT` dynamically; the app honors it
- Repo: `github.com/1903978-prog/compplan`, branch: `master`

### GitHub Actions
- `backup-db.yml` — nightly 02:00 UTC, `pg_dump` → S3, prunes (7 daily / 4 weekly / 12 monthly)
- `backup-code.yml` — nightly `git bundle` → S3

## 9. Run From Scratch (local)

```bash
# 1. Clone
git clone https://github.com/1903978-prog/compplan.git
cd compplan

# 2. Install
npm install

# 3. Start Postgres (local install or Docker)
# Docker:
#   docker run -d -p 5432:5432 -e POSTGRES_PASSWORD=postgres postgres:15
createdb compplan

# 4. Create .env in project root
cat > .env <<'EOF'
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/compplan
ANTHROPIC_API_KEY=sk-ant-...         # optional, needed for AI features
APP_PASSWORD=changeme                # optional; unset = open dev access
PORT=5000
NODE_ENV=development
EOF

# 5. Apply schema (seed also runs on first boot)
npm run db:push

# 6. Run
npm run dev
# open http://localhost:5000
```

### Minimum env vars for production on Render
- `DATABASE_URL` (Neon pooled connection)
- `ANTHROPIC_API_KEY`
- `APP_PASSWORD`
- Optionally: `API_UNPAUSE_PASSWORD`, `HARVEST_TOKEN`, `HARVEST_ACCOUNT_ID`

`PORT` and `NODE_ENV` are handled by Render.

### Troubleshooting

| Issue | Fix |
|---|---|
| `DATABASE_URL is required` | Set `DATABASE_URL` in `.env` |
| `connect ECONNREFUSED ...:5432` | Postgres not running — start it or use Docker |
| AI endpoints return placeholder text | Set `ANTHROPIC_API_KEY` |
| `401 Unauthorized` on API calls | Log in via `POST /api/auth/login` or unset `APP_PASSWORD` for local dev |
| Port 5000 in use | Change `PORT` in `.env` |
