## SECTION E ‚Äî HOSTILE AUDITOR REPORT (Agent 3)

### Where Agent 1 Was Too Optimistic

1. **"Not proven deployed" retracted by Agent 2 runtime evidence.** Agent 1 correctly flagged all 4 as unproven deployed. Agent 2 (browser + direct nav) confirmed deployment for R1, R2, R3, R4 live. Agent 1's caution was appropriate; the evidence was gathered by Agent 2.

2. **Active Projects = 0 was underplayed.** Agent 1 noted it as a "narrow edge case" in the cache guard. The real issue is more structural: `Active Projects` counts `won_projects` table rows; `Ongoing Projects` counts `pricing_proposals` with `outcome='won'` and future `end_date`. Two different data models for "active work." Live shows 0 for Active Projects even though 2 projects are ongoing. This is confusing UX, not a cache bug.

3. **`markProjectOutcome` only sends `{project_name, outcome}` to PUT.** This is technically correct ‚Äî the route accepts partial updates. But the function creates a NEW proposal record via `POST /api/pricing/proposals` AND stamps the case via `PUT /api/pricing/cases/:id`. The proposal creation (line 2367) uses a hardcoded `weeklyGrossAdmin` calculation that may not match the case's canonical net value if `manualDelta` or `adminFeePct` differ from defaults. This is a latent pricing calculation inconsistency, not a wiring bug.

### Attacks on Agent 2 Findings

4. **Persistence of Templates not live-tested.** Agent 2 confirmed the "Add template" button is present but did not actually add a template, blur, and reload to confirm it saves to the DB and survives a refresh. The code path is complete (PUT ‚Üí Drizzle ‚Üí JSONB column), but end-to-end persistence is UNPROVEN by runtime evidence.

5. **Auth: browser has active session cookie ‚Äî not evidence auth works.** The app loaded without a login prompt because the browser had a valid 30-day `compplan_session=authenticated` cookie. A fresh incognito browser would hit the login wall. This is NOT a security hole ‚Äî it is working as designed. But it means the pre-flight "no login prompt" observation is not evidence that auth is disabled.

### Specific Weaknesses and Edge Cases

6. **`wonDisplayName` regex `replace(/[A-Z]$/, "")` mutates any project code ending in an uppercase letter.** For codes like "SCHA" (if it ever existed without a numeric suffix), this strips the last letter of the project name itself. Safe for current Eendigo naming convention (codes end in digits + revision letter), but fragile as a permanent fix.

7. **`writeCache` guard (line 173) is asymmetric.** The condition is `employees.length === 0 && candidates.length === 0 && wonProjects.length === 0`. If `employees` has data but `wonProjects` is empty (valid: no won deals yet), the cache IS written with `wonProjects: []`. On next load, Active Projects will correctly show 0 from cache ‚Äî but if the API call to `/api/won-projects` fails on the next visit, the cached `[]` will remain shown as "0 Active Projects" indefinitely. Not a bug but a design tension.

8. **`templates` array has no size cap in the org-agents PUT handler.** Lines 912‚Äì914 cap `goals` (30), `okrs` (10), `tasks_10d` (50) ‚Äî but `templates` has no cap. A user could POST a large `templates` array. Minor JSONB-bomb risk.

9. **Session value is `"authenticated"` ‚Äî a fixed string.** Any browser that manually crafts `compplan_session=authenticated` bypasses auth without knowing `APP_PASSWORD`. The cookie is `httpOnly: true` so JS can't read it, and `secure: true` in production (HTTPS only) makes it inaccessible over HTTP. Theoretical risk for an attacker with network access, but very low severity for a single-user internal tool. No cryptographic signing (JWT/HMAC) is used.

10. **Version label `v24Mar` is hardcoded in App.tsx:344.** Cannot use this to verify which commit is deployed. The only reliable version verification is the Render Events deploy hash.

11. **`/org-chart` direct URL returns 404.** The page lives at `/exec/org-chart` (confirmed by browser URL). Any external link or bookmark to `/org-chart` breaks. This is a navigation/link finding (Dimension C).

### False Positives Corrected

- **Auth not active**: FALSE ‚Äî auth IS active. Browser had a valid 30-day session cookie.
- **Previous commits not deployed**: FALSE ‚Äî Render auto-deploys on push; all commits confirmed deployed via Render Events.
